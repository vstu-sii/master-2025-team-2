# Stakeholder Analysis

## 1. Primary Stakeholders (Основные заинтересованные стороны)

### Радиологи и рентгенологи
Ключевые пользователи системы, непосредственно работающие с рентгеновскими изображениями. Они формируют клинические заключения, и для них система снижает нагрузку, ускоряет работу и повышает точность. Их мнение критически важно для принятия решений о функциональности модели и точности описаний.

### Врачи общей практики и клинические специалисты
Потребители результатов, которые используют сгенерированные заключения для постановки диагноза и выбора стратегии лечения. Для них важна ясность, структурированность и клиническая применимость выводов модели.

### Медицинские центры, клиники и диагностические отделения
Организации, внедряющие систему в рабочие процессы. Они заинтересованы в скорости обработки снимков, сокращении затрат и улучшении качества обслуживания пациентов. Руководители клиник определяют бюджет, сроки внедрения и масштабирование.

---

## 2. Secondary Stakeholders (Вторичные заинтересованные стороны)

### Пациенты
Хотя они не взаимодействуют напрямую с системой, конечный результат — скорость и качество диагностики — влияет на их медицинский опыт.

### Регуляторы и органы контроля (Росздравнадзор и др.)
Следят за соответствием медицинского ПО стандартам качества, безопасности и хранения данных. Их требования определяют юридические рамки продукта.

### Страховые компании (опционально)
Могут участвовать, если система внедряется в клинике с ДМС. Их интерес — снижение затрат за счёт более точной и быстрой диагностики.

### Разработчики LLM и исследовательские команды
Работают над улучшением модели, точности интерпретаций, безопасностью и обработкой медицинских данных. Они обеспечивают технологическую основу продукта.

## 3. Influence Level (Уровень влияния)

| Стейкхолдер | Уровень влияния |
|-------------|-----------------|
| Радиологи / рентгенологи | Очень высокий — определяют клиническую точность и жизнеспособность системы |
| Врачи других специальностей | Средний — формируют требования к понятности выводов |
| Клиники и руководители | Очень высокий — принимают решения о внедрении и финансировании |
| Пациенты | Низкий — потребители результата |
| Регуляторы | Высокий — определяют возможность коммерческого использования |
| Страховые компании | Средний — влияют на экономическую модель внедрения |
| Разработчики LLM | Высокий — определяют технологическую траекторию продукта |

| Стейкхолдер | Уровень заинтересованности |

## 5. Communication Strategy (Стратегия взаимодействия)

### Для радиологов и рентгенологов
- Регулярные UX-интервью и тестирование новых функций.  
- Вовлечение в оценку качества генерируемых заключений.  
- Создание экспертной группы для обратной связи.

### Для клиник и руководителей
- Презентация бизнес-эффекта: скорость, экономия, пропускная способность.  
- Пилотные внедрения с отчётами по эффективности.  
- Поддержка SLA и документирование процессов.

### Для врачей других специальностей
- Обучающие материалы: как интерпретировать ответы модели.  
- Краткие гайды по использованию интерфейса.  
- Сбор обратной связи по понятности описаний.

### Для пациентов
- Ненавязчивая коммуникация через клинику: повышение точности и скорости диагностики.  
- Упоминание в информационных буклетах или сервисах клиники.

### Для регуляторов
- Подготовка документации о соответствии стандартам.  
- Участие в проверках, аудитах, сертификациях.  
- Прозрачность алгоритмов, данные о безопасности и точности.

### Для страховых компаний
- Презентация данных о снижении затрат и ускорении обработки случаев.  
- Пилотные программы с ограниченными сценариями.

### Для разработчиков LLM
- Постоянный обмен данными об ошибках и сложных кейсах.  
- Планирование roadmap с учётом медицинских требований.
