# RACI Matrix — Проект: "Система описания рентгеновских снимков с использованием LLM"

---

## 1. Роли
- **SA/PO** — System Architect / Product Owner. Проектирует архитектуру всей системы, управляет требованиями, планирует работу команды, контролирует сроки и принимает ключевые решения.
- **AI Engineer** — Специалист, который создаёт и настраивает "интеллектуальную" часть системы. Отвечает за выбор модели, промпты, RAG, анализ данных и качество результатов.
- **MLOps Engineer** - Инженер, обеспечивающий стабильную работу модели в продакшене: контейнеризация, CI/CD, деплой, мониторинг и инфраструктура.
- **Fullstack Engineer** - Отвечает за интерфейс, UX, backend API, обработку файлов, интеграцию с моделью и тестирование пользовательского опыта.

## 2. Области ответственности

### **AI/ML области**
1. Выбор и настройка LLM моделей  
2. Промпт‑инжиниринг и RAG  
3. Метрики качества модели  
4. Обработка медицинских рентгеновских изображений 
5. Валидация медицинских описаний 

### **MLOps области**
6. Docker и контейнеризация  
7. CI/CD пайплайны  
8. Мониторинг и логирование системы  

### **Fullstack области**
9. UI/UX дизайн и интерфейс  
10. Backend API разработка  
11. Тестирование UI/API  

### **Architect / PO области**
12. Архитектура системы  
13. Планирование и координация  
14. Документация и презентационные материалы  
15. Управление требованиями (работа с заказчиком)  

## 3. Обозначения
- **R** — Responsible (исполнитель)
- **A** — Accountable (ответственный за результат)
- **C** — Consulted (консультант)
- **I** — Informed (осведомлён)

## 5. Процесс принятия решений

1. **Архитектурные решения** — финальное слово за **SA/PO**.  
2. **ML‑часть (модель, промпты, RAG)** — финальное слово за **AI Engineer**.  
3. **Инфраструктура и деплой** — финальное слово за **MLOps Engineer**.  
4. **UI/API решения** — финальное слово за **Fullstack Engineer**.  
5. При конфликтах — окончательное решение принимает **SA/PO**.

---

## 6. Путь эскалации конфликта

1. **Обсуждение в команде** — попытка решить проблему совместно.  
2. **Решение SA/PO** — если команда не пришла к согласию.  
3. **Обращение к ментору/преподавателю** — финальный этап.
